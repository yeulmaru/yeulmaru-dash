## 작업 결과 보고 규칙

모든 단위 작업(기능 추가, 버그 수정, 리팩토링, 데이터 조사 등) 완료 후 자동으로 보고서 생성. 사용자가 별도 요청하지 않아도 무조건 생성.

- 회사PC: `C:\Users\황세웅\OneDrive - GS칼텍스 예울마루\DAX\_claude_output\`
- 폴더 없으면 자동 생성

### 파일명 형식
`YYMMDD_작업명_HHMMSS.md`
- 예: `260406_1단계완료_013045.md`
- 예: `260406_판매현황버그수정_142233.md`
- HHMMSS는 현재 시각 추정값
- 덮어쓰기 절대 금지, 매번 새 파일 생성
- 확장자 무조건 `.md` (모바일 Claude가 M365 SharePoint 커넥터로 읽기 위함)
- `.py`, `.json`, `.txt` 등 다른 형식 금지

### 필수 섹션 (순서 준수)
1. ## 작업 목적 (1~2줄)
2. ## 변경 파일 목록 (생성/수정/삭제 경로)
3. ## 주요 변경 내용 (before/after 표 또는 diff)
4. ## 테스트 결과 (PASS/FAIL 표)
5. ## 발생 이슈 (경고, 에러, 주의사항)
6. ## 다음 단계 (후속 작업 필요한 것)

### 코드 포함 규칙
- 코드는 ```python 블록으로 발췌만
- 전체 파일 내용 붙여넣기 금지
- 핵심 로직 10~20줄만 추출
- diff는 +/- 형식 표시

### 상태 아이콘 필수
각 섹션/항목 앞에 ✅(완료) / ⚠️(경고) / ❌(실패) 표시

### 왜 이 규칙이 필요한가
사용자가 모바일 Claude 앱에서 M365 SharePoint 커넥터로 이 보고서를 읽는다.
코드 파일(.py 등)은 M365 커넥터에서 MIME 타입 거부로 못 읽으므로 .md 보고서가 반드시 필요하다.

## 한글 인코딩 주의사항

### Edit 도구 한글 3바이트 문자 손상 버그
Edit 도구가 한글 포함 코드 수정 시 한글 3바이트 문자 일부를 깨뜨리는 현상 확인됨 (1단계 작업에서 실제 발생).

### 대응 방법
한글 포함 코드 수정 시:
1. Edit 도구 사용 금지
2. 대신 다음 순서로 처리:
   a. 파일 전체 백업
   b. Read로 원본 읽기
   c. Python 스크립트(str.replace 또는 re.sub)로 치환
   d. Write 도구로 전체 덮어쓰기
   e. 재읽기해서 한글 깨짐 검증
3. 깨짐 발견 시 즉시 백업 복원 후 재시도

이 규칙 지키지 않으면 파일 한글 주석/문자열이 깨져서 인코딩 에러 발생.
