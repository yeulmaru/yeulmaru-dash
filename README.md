# GS칼텍스 예울마루 공연 운영실적 대시보드

이 프로젝트는 예울마루의 공연 운영실적 엑셀 데이터를 바탕으로 주요 실적(당일 판매현황, 25년도 예산/지출/매출 비교, 과거부터 현재까지 연도별 실적 트렌드)을 시각화하여 보여주는 Streamlit 웹 대시보드입니다.

## 기능 요약

1. **실시간 판매현황**: `일일입력`, `판매추이` 시트를 바탕으로 현재 공연별 판매좌석, 금액, 점유율 및 판매 추이 꺾은선 차트를 시각화하여 보여줍니다.
2. **공연실적**: `25공연` 시트를 바탕으로 당해 연도의 다양한 공연들의 총 예산, 지출, 매출을 차트로 비교하며 카테고리 비율 및 수익률 랭킹을 제공합니다.
3. **연간실적**: 2012년부터 누적 기록된 `운영실적통합`, `세부운영관리대장` 시트 데이터를 바탕으로 연간/장르별 트렌드를 분석하고 필터링과 CSV 다운로드가 가능한 데이터 테이블을 제공합니다.

## 로컬 개발 및 실행

이 앱은 Python 3.8 이상 환경을 권장합니다.

```bash
# 필수 패키지 설치
pip install -r requirements.txt

# 데이터 준비
# 이 앱은 `data/` 위치에 최신 실적 엑셀 파일을 감지하고 읽어들입니다.
# data/ 폴더 내에 엑셀파일 1개를 위치시켜 주세요.

# 메인 앱 실행
streamlit run app.py
```

## GitHub 연동 및 Streamlit Cloud 배포 가이드

1. **GitHub 푸시**:
   - 현재 구현된 코드들을 모두 GitHub Repository로 PUSH 하세요. (이미 엑셀 파일이 `data/` 내에 존재하면 함께 올라갑니다.)
2. **Streamlit Cloud에 연결하기**:
   - [Streamlit Cloud](https://streamlit.io/cloud) 로그인 후 **"New app"** 클릭
   - `Repository` 에 방금 올린 저장소(Repo)를 선택하고 `Branch`를 지정합니다.
   - `Main file path` 란에 **`app.py`** 를 기입하고 Deploy를 누릅니다.
3. **추후 데이터 갱신 방법**:
   - 새로 업데이트된 `.xlsx` 실적 파일이 나오면, 로컬 `data/` 디렉토리 안의 파일을 삭제하고 새 파일로 교체하세요.
   - 그 후 GitHub에 Commit & Push 합니다.
   - Streamlit Cloud에서 변경사항을 감지하여 자동으로 대시보드를 갱신합니다. (최대 5분 캐시 TTL 대기)
